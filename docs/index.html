Stay ahead of resets.

CodexBar keeps session + weekly limits (and credits) in the menu bar, so you know when you're safe to ship.

Requirements: macOS 14+ (Apple Silicon + Intel).

Download latest View on GitHub

Homebrew: brew install --cask steipete/tap/codexbar

Linuxbrew (CLI only): brew install steipete/tap/codexbar

GitHub Releases builds update via Sparkle. Homebrew installs update via brew upgrade.

  • One status item per provider (Codex, Claude, Cursor, Gemini, Antigravity, Droid, Copilot, z.ai).
  • Session + weekly bars with reset times and optional credits.
  • Web-backed limits via browser cookies (Codex, Claude, Cursor, Droid).
  • Two-bar icon: session on top, weekly on bottom, dimmed on errors.
  • Minimal UI, no Dock icon.

Install & updates

Download from GitHub Releases or install via Homebrew. Sparkle updates are enabled on GitHub builds only.

Providers

Codex · Claude Code · Cursor · Gemini · Antigravity · Droid · Copilot · z.ai
Open to new providers — see the provider guide.

Privacy

Reuses existing browser cookies (Safari/Chrome/Firefox) for dashboard access — no passwords stored. When cookies are missing, CodexBar falls back to local CLI output. Audit notes in issue #12.

Usage tracking

Session + weekly meters with reset countdowns, plus optional credits.

Status + widgets

Provider status polling with incident badges, plus a WidgetKit snapshot that mirrors the menu card.

CLI + automation

Bundled codexbar CLI for scripts and CI, with Linux CLI builds on GitHub Releases.

macOS permissions

Full Disk Access only for Safari cookies, Keychain access for tokens, and Files & Folders prompts when provider CLIs access project directories.
